# LOGBOOK — каркас приложения

Это рабочий скелет, а не законченный продукт: офлайн-хранилище и сканирование QR
уже работают "из коробки", Supabase / AI / перевод — подключаются по мере готовности.

## Что уже работает без всякой настройки

1. Откройте `index.html` через локальный сервер (не через `file://` — Service Worker
   и камера требуют http/https). Проще всего:
   ```
   npx serve .
   ```
   или в VS Code — расширение "Live Server".
2. Откроется список оборудования с одним предзаполненным котлом (`53817R1`).
3. Кнопка "📷 Сканировать QR" запросит доступ к камере и распознает QR-код
   через библиотеку jsQR (работает и в Safari на iPhone, в отличие от
   нативного `BarcodeDetector`, который есть только в Chrome/Android).
4. Всё, что вы сохраните (записи в журнал, загруженные документы —
   как только настроите Supabase), останется в IndexedDB браузера —
   переживёт закрытие вкладки и офлайн.

## Шаг 1 — Разместить в интернете (нужно для HTTPS и "Добавить на экран")

Самый простой бесплатный вариант — GitHub Pages:
1. Создайте репозиторий на GitHub, загрузите туда все файлы этой папки
2. Settings → Pages → Source: выберите ветку `main`, папку `/root`
3. Через 1-2 минуты приложение будет доступно по `https://<ваш-логин>.github.io/<репозиторий>/`
4. Откройте эту ссылку на телефоне → в браузере выберите "Добавить на экран Домой"

Альтернативы: Netlify, Vercel, Cloudflare Pages — все дают бесплатный HTTPS-хостинг
статики за пару кликов, просто перетащить папку в веб-интерфейс.

## Шаг 2 — Подключить Supabase (общая база для нескольких механиков)

1. Зарегистрируйтесь на https://supabase.com, создайте новый проект
2. В интерфейсе слева: SQL Editor → New query → вставьте содержимое
   `supabase_schema.sql` → Run
3. Settings → API → скопируйте `Project URL` и `anon public` ключ
4. Вставьте их в `app.js`, в объект `CONFIG` вверху файла:
   ```js
   SUPABASE_URL: 'https://xxxxxxxx.supabase.co',
   SUPABASE_ANON_KEY: 'eyJhbGci...'
   ```
5. Authentication → Providers → включите "Email" (magic link, вход без пароля)
6. Authentication → URL Configuration → добавьте адрес вашего хостинга (шаг 1)
   в Site URL и Redirect URLs — иначе ссылка из письма не вернёт человека в приложение
7. Storage → New bucket → назовите `documents`, включите "Public bucket"
   (либо оставьте приватным и настройте signed URLs — тогда `getPublicUrl`
   в `app.js` нужно заменить на `createSignedUrl`)
8. Storage → бакет `documents` → Policies → разрешите `authenticated` роли
   `INSERT` и `SELECT` (по аналогии с политиками в SQL выше — Storage
   политики настраиваются отдельно от таблиц, через интерфейс Supabase)

Авторизация уже подключена в UI (`view-login`, `view-setname` в `index.html`,
логика в `app.js`): человек вводит email, получает magic link на почту,
при первом входе указывает имя. Дальше сессия хранится в браузере и работает
офлайн.

**Регистрация сейчас полностью открытая** — зарегистрироваться может любой,
кто знает адрес приложения и имеет email. Если нужно ограничить доступ только
своими людьми — отключите публичную регистрацию в Supabase (Authentication →
Providers → Email → "Allow new users to sign up" выключить) и добавляйте
пользователей вручную через Authentication → Users → Invite.

После этого `Data.*` в `app.js` начнёт автоматически синхронизировать записи
в облако при наличии интернета и продолжит работать локально без него —
переключение уже реализовано в коде (`_syncOrQueue`, `flushQueue`).

**Важно про anon-ключ:** он специально считается публичным и его можно
хранить в клиентском коде — безопасность обеспечивается Row Level Security
(политики в конце `supabase_schema.sql`), а не секретностью ключа. Не путайте
его с `service_role` ключом — тот вставлять в приложение нельзя никогда.

## Шаг 3 — AI-анализ схемы (нужен свой backend, не вызывать API напрямую из браузера)

Почему нельзя просто вписать ключ Anthropic в `app.js`: он будет виден любому
в исходном коде страницы и его украдут за часы. Нужен небольшой backend-прокси.

**Важно про содержимое PDF:** сейчас `app.js` передаёт в `AI_ENDPOINT` только
список документов (название, ссылка, кто загрузил) — сам текст/чертёж внутри
PDF в контекст не попадает. Чтобы AI реально анализировал содержимое схемы, а
не просто видел список файлов, Edge Function ниже должна дополнительно
скачать сами PDF по `file_url` и передать их в Anthropic API как `document`
content-блоки (Claude умеет читать PDF напрямую):

```ts
// supabase/functions/ask-ai/index.ts
import Anthropic from "npm:@anthropic-ai/sdk";

Deno.serve(async (req) => {
  const { equipment_id, question, context } = await req.json();
  const anthropic = new Anthropic({ apiKey: Deno.env.get("ANTHROPIC_API_KEY") });

  // Скачиваем сами PDF и кодируем в base64, чтобы Claude мог их прочитать,
  // а не просто видел названия файлов.
  const pdfBlocks = [];
  for (const doc of [...context.manualDocs, ...context.schemaDocs]) {
    const resp = await fetch(doc.file_url);
    const buf = await resp.arrayBuffer();
    const base64 = btoa(String.fromCharCode(...new Uint8Array(buf)));
    pdfBlocks.push({ type: "document", source: { type: "base64", media_type: "application/pdf", data: base64 } });
  }

  const msg = await anthropic.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 1000,
    messages: [{
      role: "user",
      content: [
        ...pdfBlocks,
        { type: "text", text: `Ты помогаешь электромеханику на судне. Записи журнала по этой системе:
${JSON.stringify(context.journal)}

Вопрос: ${question}

Отвечай конкретно, со ссылками на листы/теги/клеммы из документов. Если
данных недостаточно для точного ответа — так и скажи, не выдумывай.` }
      ]
    }]
  });

  return new Response(JSON.stringify({ answer: msg.content[0].text }), {
    headers: { "Content-Type": "application/json" }
  });
});
```

Ограничение: если документов/страниц много, это быстро упрётся в лимит
контекста и в стоимость запроса — для больших архивов чертежей понадобится
предварительная индексация (примерно то, что мы вручную сделали для этого
котла в начале разговора — вытащить структуру в JSON один раз, а не гонять
весь PDF при каждом вопросе).

Задеплоить: `supabase functions deploy ask-ai`, ключ хранить через
`supabase secrets set ANTHROPIC_API_KEY=...` (никогда не в коде).

Затем в `app.js` укажите:
```js
AI_ENDPOINT: 'https://xxxxxxxx.supabase.co/functions/v1/ask-ai'
```

## Шаг 4 — Перевод мануала/схемы

Это стало сложнее, чем в версии с текстовыми полями: теперь нужно
переводить содержимое PDF, а не готовый текст. Практичный путь:

1. Edge Function скачивает PDF, извлекает текст (например, через
   `unpdf` или аналогичную библиотеку, доступную в Deno)
2. Переводит текст через DeepL API
3. Возвращает переведённый текст (показать инлайн в приложении) —
   **не** пытайтесь на этом этапе генерировать новый PDF, это отдельная
   и более дорогая задача (вёрстка, изображения, таблицы)
4. При желании сохранить результат — загрузите переведённый текст как
   новый документ (`doc_type: 'manual'`, title вида "EN translation of ...")
   через тот же `Data.uploadDocument`, только с текстовым файлом вместо PDF

Для полноценного PDF-в-PDF перевода (с сохранением вёрстки) потребуется
более серьёзный инструмент уровня Adobe API или специализированный сервис —
не стоит делать это в MVP.

## Шаг 5 — Генерация QR-кодов для щитов

Любой генератор QR (например, `qrcode` npm-пакет или онлайн-сервис) —
закодируйте туда просто код оборудования, например `53817R1`. Распечатайте
на промышленной плёнке (полиэстер, устойчив к маслу/влаге), приклейте
рядом с заводской табличкой.

## Мануал и схема теперь — PDF-документы, а не редактируемый текст

Решение изменилось: оригинальный мануал и электросхема загружаются как PDF
и **никогда не перезаписываются**. Любой авторизованный механик может:
- загрузить оригинальный мануал/схему, если их ещё нет
- добавить ещё один PDF поверх существующих (исправление, доработка,
  перевод, скан рукописной пометки — как лист 8 в вашей реальной схеме)

Но никто не может отредактировать или удалить уже загруженный файл — в
`supabase_schema.sql` для таблицы `documents` **намеренно не созданы**
политики `update`/`delete`, поэтому Postgres запрещает это на уровне базы,
а не только в интерфейсе (интерфейс можно обойти, RLS — нет).

Список документов растёт со временем: пять человек могут загрузить пять
версий/дополнений одного мануала, и все пять останутся видны с именем
загрузившего и датой — это и есть история, только не как единая
изменяемая запись, а как последовательность файлов.

**Офлайн-ограничение:** загрузка PDF требует подключения к интернету —
бинарные файлы не встают в офлайн-очередь синхронизации в этом каркасе
(в отличие от текстовых записей журнала). Если механик в машинном
отделении без сети — кнопка загрузки покажет предупреждение и попросит
подключиться к интернету.
